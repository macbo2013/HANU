# HANU V10 正式版

版本号：`1.0.2`  
建议 Release Tag：`v1.0.2`

## 本次定位

HANU V10 是从公测版进入正式版的整理发布版本。

本次重点不是单独新增一个小功能，而是把 V9 期间出现的零散问题统一修复，并加入完整的积分名称自定义和积分红包系统。

## 主要更新

### 1. 合并已知热修复

V10 已合并以下修复：

- 修复 `/app/update.php` 缺少 `layout.php` 导致 `shell()` 不存在的问题
- 修复 `/app/about.php` 缺少 `update.php` 导致 `hanu_current_version()` 不存在的问题
- 修复 `/app/banned.php` 缺少 `layout.php` 导致 `page_head()` 不存在的问题
- 修复更新中心函数重复声明问题
- 修复群聊发消息稳定性
- 群聊增加密码保护
- 板块列表改为按钮进入
- 删除板块页面多余提示文字

### 2. 新增积分名称自定义

管理员可以在后台设置积分显示名称。

路径：

```text
/app/admin.php
```

可将积分改名为：

```text
金钱
金币
贡献值
经验
社区币
```

说明：

- 这里只改变显示名称
- 不直接修改用户余额
- 原有积分数据继续兼容

### 3. 新增积分红包系统

新增页面：

```text
/app/red_packets.php
```

用户可以：

- 发积分红包
- 抢积分红包
- 查看最近红包
- 查看最近领取记录

红包规则：

- 发送红包会立即扣除发送者积分
- 领取红包会自动增加领取者积分
- 同一个红包同一个用户只能领取一次
- 不能领取自己发的红包
- 红包领完后自动变为完成状态

### 4. 保留更新源设置

V10 仍然保留后台更新中心的：

```text
更新源设置
```

路径：

```text
/app/update.php
```

管理员仍可设置：

```text
GitHub 仓库
分支
```

更新不是强制更新，只有管理员可以看到更新提醒。

### 5. 数据库兼容升级

V10 数据库迁移只会：

```text
新增缺失表
新增缺失字段
写入必要设置
```

不会：

```text
清空用户
删除旧表
重建数据库
覆盖 config/config.php
删除 data/
```

## 升级方式

服务器进入网站根目录：

```bash
sudo bash update.sh
```

然后运行：

```bash
php cli/migrate.php
```

或者进入后台：

```text
/app/update.php
```

点击：

```text
运行数据库兼容升级
```

## 保留目录

更新时必须保留：

```text
config/config.php
data/
ICO/
```
